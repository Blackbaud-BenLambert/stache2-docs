While we try prevent issues by creating exceptional documentation, we know you may run into problems from time-to-time. On this page, learn how to resolve them. Based on feedback from you and trends we notice, we'll continue to update our documentation and add notes here. Let us know how we can help you better. NPM Install

Scenarios

  • When you were running skyux new --template stache or npm install, did the install time out?

  • Are you having a problem serving your site locally?

  • Did you update the dependencies in the package.json file?

Solution

You may want to re-run npm install. Before you do that, we recommend you delete your entire node_modules folder. In the command line for the Stache site folder, enter npm cache clear. When that completes, enter npm install.

Page isn't loading locally

Scenario

When you run skyux serve, does a browser window open to your "in-dev" site, but all you see is a bouncing green ball and the page doesn't load?

Solution

When this happens, it means there's an error in the code for your Stache site. We recommend you use the browser developer tools to find the error. For example, in Chrome, right-click on the page and select Inspect. Then, select the Console tab. Do you see a lot of red text? Scroll up to the top to see if you can find a clue as to where you made a mistake in the code.

Common issues we've noticed:

  • Did you provide an opening tag, like <stache-page-anchor> but not the closing one </stache-page-anchor>?
  • Check your json files for errors. These can include an unnecessary comma, missing comma, unncessary space, unncessary tab, missing character ( {, [, }, ], etc), and improper formatting.
  • Did you reference an image that hasn't been added to your assets folder yet?
  • Did you misspell a component? If you reference the stace-code, it will actually go try to find that component and fail.

If you've made a recent change, but everything looks good, you can also stop serving and serve again to see if that fixes the issue. If it didn't, keep looking, as there's an error in there somewhere. Still can't find it? Contact us and we'd be glad to help hunt it down.
