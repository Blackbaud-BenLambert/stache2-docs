Stache is brought to you by the User Education team. We constantly improve content development for the benefit of all consumers, our customers and you.

Features

- Component library for SKY UX SPAs, which means all the goodness of SKY UX plus features customized for documentation - Tutorial pattern - Inline code and code block - Blockquote - Table of Contents to provide in-page navigation - Easy to embed images and videos - Grid so you can design a page into responsive columns - Fully responsive experience - Source control with Git, do you want the ability to review content before it's live? - Templated - Documentation layouts - 2 options for who can view your site? Blackbaud users only with Blackbaud ID or public. Sites can even start internal and be made public later. - Hosted in Azure - Stache Search Service - Search within a Stache Site - Search can include a group of Stache sites - Be a part of the global search, which searches through all search-enabled Stache Sites - You can exclude yourself from global search, if needed - Quick SPA and VSTS repo set-up with the SKY UX App Management Portal - With a few simple commands, you can have it set up locally on your computer - No need to know Angular, you can document in markdown or html - Open source, accepts contributions - Extensible, because it's a SKY UX app, you can build out custom features you need or even mix tooling and docs like Engineering System - Automatically configured deployments - Because Stache is a component library for a SKY UX SPA, you get all the testing and deployment features - No need to install locally, you can edit online in VSTS web view - Stache works with the new omnibar - Smart features for advanced documentation - global JSON data variables, so you can define a name, links, phrases, etc, then reference it throughout your site. If it ever changes, you only need to update in one place. - includes, create content in one file that you can include in many. - optionally, can add custom css - VSTS repos are accessible to anyone in RDO - Open source projects can be housed in GitHub for even greater opportunity for contributions - Stache is in-progress, for the benefit of us all. Which means we are here to adapt and add features that your content development experts (User Ed) and you, the community, request. End result is with our documentation patterns and components coupled with SKY UX, you have the ability to create a well-designed documentation experience, consistent across RDO. However, Stache isn't for early collaborative documentation or projects where a team is taking a lot of notes and working through a solution together. When you need to: - share knowledge, including code, across a wide group - teach - onboard - raise awareness - tooling help content - optimize your content for an external audience the time you invest in creating good documentation can result in less friction for your users learning and understanding. Enable efficient self-service so you can spend less time answering questions.
